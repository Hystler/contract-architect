# Contract Architect Design System

Источник правды для редизайна MVP. Сформирован по `contract_architect_ai_redesign_spec.md` и workflow `ui-ux-pro-max-skill`.

## Продуктовый принцип

Contract Architect — не AI-юрист. Это премиальный legal-tech workspace: шаблонный генератор договоров с AI-помощником, который объясняет текст, подсвечивает потенциально спорные места и помогает подготовить вопросы к специалисту.

## Визуальное направление

- Стиль: premium legal-tech workspace.
- Ощущение: контроль, точность, спокойствие, документная ясность.
- Hero: тёмный matte-фон с крупной рабочей поверхностью документа.
- Рабочая зона: светлая бумажная поверхность, графитовая типографика, аккуратные границы.
- AI-слой: спокойный Blue Intelligence, без фиолетово-розовых SaaS-градиентов.
- Не использовать фейковые отзывы, клиентов, обещания юридической силы или формулировки “заменяет юриста”.

## Типографика

- Заголовки: Cormorant Garamond, fallback Georgia/serif.
- Интерфейс: Manrope, fallback system sans.
- Документный preview: Times New Roman/serif.
- Базовый размер текста: не ниже 16px на мобильных.
- Для длинных реквизитов использовать `break-words`, `whitespace-pre-line`, ограниченную ширину строки.

## Компоненты

- Радиус карточек и контролов: 8px или меньше.
- Кнопки: один главный CTA на экран; secondary действия визуально тише.
- Поля: всегда видимые label, ошибка под конкретным полем, focus ring видимый.
- Карточки: только для самостоятельных объектов, панелей и повторяющихся элементов. Не вкладывать карточки в карточки.
- AI-предложения показывать отдельно от текста договора. Не применять автоматически.

## Builder layout

Desktop:

1. Левая колонка: структура сценария и выбранный шаблон.
2. Центр: форма/рабочая область.
3. Правая sticky-колонка: резюме документа и AI-помощник.

Mobile:

- Колонки складываются вертикально.
- Основная форма идёт первой после выбора шаблона.
- AI-панель становится обычной секцией ниже preview/резюме.
- Все touch targets не меньше 44px.

## Motion

- Только мягкие transition/fade/slide в пределах 150-300ms.
- Не блокировать ввод анимациями.
- Уважать `prefers-reduced-motion`.
- Hover не должен менять размер layout.

## AI safety copy

Обязательный дисклеймер:

> AI-помощник помогает объяснить текст и обратить внимание на возможные спорные места. Это не является юридической консультацией и не заменяет проверку специалистом.

## Anti-patterns

- Не писать “договор без рисков”, “AI гарантирует безопасность”, “замена юриста”.
- Не использовать кислотные цвета, фиолетово-розовый SaaS look, дешёвые бейджи.
- Не использовать emoji как структурные иконки.
- Не перегружать генератор декоративными эффектами.
- Не отправлять AI-ключи или полный внутренний debug на клиент.

## QA

- Проверить `/`, `/generator`, `/preview`.
- Проверить DOCX/ZIP экспорт через `/api/generate`.
- Проверить AI route без ключа: приложение не падает, показывает понятную ошибку.
- Проверить AI route с ключом: explain, risk hints, rewrite, questions.
- Проверить mobile 375px, tablet 768px, desktop 1440px.
- Запустить lint, typecheck, build.
